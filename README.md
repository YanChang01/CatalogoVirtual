# CatalogoVirtual

Esta es una web diseñada como catálogo virtual para un negocio de ventas de SexToys

## Como usar

### Frontend

#### Requisitos

- Node.js 24.12.0 LTS
- npm 11.6.2 (debe venir con el node 24.12.0 LTS)

#### Instalar dependencias

En la carpeta `frontend` abrir la terminal de comandos y ejecutar:
`npm install`.

#### Ejecutar

En la carpeta `frontend` abrir la terminal de comandos y ejecutar:
`npm run dev`.

Luego de que se ejecute correctamente el comando, va a salir en consola el servidor de Vite. Para abrir la página en el navegador escribe la letra `o` y presiona `Enter`. Para abrir la ayuda escribe `h` y presiona `Enter`. Para cerrar el servidor escribe `q` y presiona `Enter` (también sirve presionar `Ctrl + C`).

### Backend

Instrucciones y Requerimientos para implementar el backend
1- Descargar e instalar Python 3.14.2
2- Descargar e instalar una versión reciente de PostgreSQL y PgAdmin 4, preferentemente las versiones 17 y 9.14 respectivamente.
3- Descargar e instalar una versión reciente de Visual Studio Code, acordar con el líder del proyecto cuales son las extensiones adicionales que se necesitan descargar e instalar.
4- Descargar e instalar Git para el control de versiones y la colaboración en equipo.
5- Utilizando PgAdmin 4 crear una base de datos PostgreSQL, acordar con el líder del Proyecto que nombre debe tener.
6- Abrir una terminal en VScode y posicionarse en la carpeta backend del proyecto.
7- Crear un entorno virtual de Python con el siguiente comando: "python -m venv env", esto debe crear una carpeta env dentro de la carpeta backend.
8- Activar el entorno virtual. Posicionarse en la carpeta backend y ejecutar el siguiente comando:
".\env\Scripts\activate". Si se activa el entorno virtual correctamente debe salir, al principio de la terminal, (env) en letras verdes.
9- Instalar todas las dependencias del archivo requirements.txt. Para ello, con el entorno virtual previamente activado, ejecutar el siguiente comando: "pip install -r requirements.txt"
10- Posicionarse en la carpeta backend y crear un archivo .env que contenga las variables de entorno. Acordar con el líder del proyecto cuales van a ser esas variables y que valores deben tener.
